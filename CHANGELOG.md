# Changelog — tender-scraper

## 1.1.0 — 2026-06-12 — надійність, AI-шар, Австрія
- **Нове джерело: USP Bekanntmachungsportal (AT)** — офіційний агрегатор австрійських
  Kerndaten за BVergG 2018, включно з підпороговими тендерами, яких немає в TED
  (`--source usp-at`, терміни в `config.json:usp_terms`).
- **AI-шар у дайджесті** (Level 2/3 з README тепер реально працює): за наявності
  `ANTHROPIC_API_KEY` (claude-haiku-4-5) або `OPENAI_API_KEY` топ-тендери отримують
  пояснення релевантності українською в Telegram + чернетки outreach-листів німецькою
  в `output/tender_outreach_*.md`. Без ключа — поведінка як раніше. `--no-ai` вимикає.
- **Health-моніторинг джерел**: статистика fetched/kept по кожному джерелу в
  `.state.json:source_stats`; рядок «📡 …» у Telegram-дайджесті; алерт «⚠️ джерело
  N запусків підряд без результатів» — тихі смерті HTML-скрейперів тепер видно.
- **Нагадування про дедлайни**: відомі тендери зі score ≥ 70 і дедлайном ≤ 7 днів
  повторно показуються в дайджесті в блоці «🔚 Згорають».
- **Ваги скорингу винесено в `config.json:scoring`** (дефолти в коді) — обіцянка
  README «код не чіпаємо» тепер стосується і скорингу.
- **TTL для `.state.json`**: записи старші 540 днів чистяться автоматично
  (раніше стан ріс безмежно).
- **Кращий витяг бюджету TED**: пріоритет procedure/global над лотами, сума по
  багатолотових тендерах (раніше брався лише перший лот), додано поле
  `estimated-value-glo` у запит.
- `tenders_<region>_latest.csv` тепер оновлюється при кожному дефолтному запуску
  (раніше застигав).
- Фікс: краш дайджесту на порожньому `priority_score`.
- Обидва скрипти: `__version__`, `--version`, версія у футері Telegram-повідомлень;
  digest отримав `--no-telegram`; захист від ліміту Telegram 4096 символів.
- Рефакторинг: 13 ідентичних блоків джерел у `main()` згорнуто в реєстр + `consume()`.
